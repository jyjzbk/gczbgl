🔧 核心功能特性

  • 分级配置管理：支持省、市、区县三级组织的实验要求配置
  • 实验类型区分：分别配置分组实验和演示实验的要求
  • 灵活的数量要求：可设置图片和视频的最少、最多数量要求
  • 继承机制：下级可选择继承上级配置或独立配置
  • 权限控制：基于组织层级的权限控制，确保数据安全
  • 有效配置计算：自动计算并显示实际生效的配置

📊 数据库结构

CREATE TABLE experiment_requirements_config ( id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY, organization_type ENUM('province', 'city', 'county') COMMENT '组织类型', organization_id BIGINT UNSIGNED COMMENT '组织ID', experiment_type ENUM('分组实验', '演示实验') COMMENT '实验类型', min_images INT DEFAULT 0 COMMENT '最少图片数量', max_images INT DEFAULT 10 COMMENT '最多图片数量', min_videos INT DEFAULT 0 COMMENT '最少视频数量', max_videos INT DEFAULT 3 COMMENT '最多视频数量', is_inherited BOOLEAN DEFAULT TRUE COMMENT '是否继承上级配置', created_by BIGINT UNSIGNED COMMENT '创建人', description TEXT COMMENT '配置说明', is_active BOOLEAN DEFAULT TRUE COMMENT '是否启用', created_at TIMESTAMP, updated_at TIMESTAMP );

🎯 访问方式

前端页面访问：

http://localhost:3000/experiment-requirements-config

API接口基础路径：

http://localhost:8000/api/experiment-requirements-config

👥 权限说明

  • 省级管理员：可以管理省级配置，查看所有配置
  • 市级管理员：可以管理本市配置，查看省级和本市配置
  • 区县级管理员：可以管理本区县配置，查看省级、市级和本区县配置
  • 学校用户：只能查看有效配置，不能创建或修改
